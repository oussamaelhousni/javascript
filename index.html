1. Introduction à JavaScript

  • Qu'est-ce que JavaScript ?
    JavaScript est un langage de programmation utilisé pour rendre les pages web interactives et dynamiques.
  • Pourquoi les entreprises l'utilisent ?
    Parce qu'il fonctionne sur tous les navigateurs, est rapide, flexible et permet de créer des applications modernes.
  • Standard et évolution : le langage suit le standard ECMAScript (TC39) ; les moteurs (V8 dans Chrome, SpiderMonkey dans Firefox, JavaScriptCore dans Safari) l’implémentent.

JavaScript, HTML et CSS : rôles clairs

Histoire rapide

JavaScript a été créé en 1995 par Brendan Eich chez Netscape Communications. Son objectif était de rendre les pages web interactives sans recharger la page. Aujourd'hui, JavaScript est utilisé par des millions d'entreprises dans le monde.

HTML

Langage de structure.
Définit le contenu de la page : titres, textes, images, formulaires.

Exemple : titre, bouton, formulaire

CSS

Langage de présentation.
Gère l'apparence visuelle : couleurs, tailles, alignement, responsive design.

Exemple : design, animations, layout

JavaScript

Langage de programmation.
Gère le comportement : clics, calculs, logique, communication serveur.

Exemple : panier e-commerce, validation de formulaire

Ce que JavaScript peut faire

  • Sites e-commerce (panier, paiement, filtres produits)
  • Applications web (Facebook, Gmail, Google Maps)
  • Validation des formulaires en temps réel
  • Animations et interactions utilisateur
  • Communication avec des serveurs (API)
  • Autres usages : applications mobiles (React Native), bureau (Electron), objets connectés, automatisation (Node.js), et PWA (sites installables comme des apps).

3. Fonctionnement de JavaScript dans le navigateur

  • Le navigateur lit et interprète les fichiers HTML, CSS et JavaScript.
  • Chaque type de fichier a un rôle précis dans le chargement de la page.
  • Comprendre cet ordre évite beaucoup d'erreurs courantes.

Comment le navigateur lit HTML, CSS et JavaScript

  1. HTML : le navigateur construit la structure de la page (DOM).
  2. CSS : il applique le style (couleurs, positions, tailles).
  3. JavaScript : il exécute le code pour ajouter du comportement.

Si JavaScript est exécuté trop tôt, les éléments HTML peuvent ne pas exister.

La balise <script>

Où placer la balise script ?

  • En bas du <body> (recommandé pour débutants)
  • Permet au HTML d'être chargé avant l'exécution de JavaScript
  • Évite les erreurs liées aux éléments non encore chargés 
<body>
  <h1>Ma page</h1>

  <!-- Exécution après analyse du HTML : -->
  <script src="script.js" defer></script>
</body>
  • defer : télécharge en parallèle, exécute le script après le HTML (ordre conservé si plusieurs scripts).
  • async : télécharge en parallèle, exécute dès que prêt (ordre non garanti — plutôt pour scripts indépendants).

Scripts inline vs fichiers externes

Script inline

Code JavaScript écrit directement dans le HTML. 

<script>
alert("Bonjour");
</script>

À éviter pour les grands projets

Fichier externe

Code JavaScript dans un fichier séparé. 

<script src="script.js"></script>

Meilleure pratique professionnelle 

<script type="module" src="app.js"></script>

Modules ES : portée par fichier, import / export, chargement différé comme defer.

Ordre d'exécution et erreurs courantes

  • JavaScript est exécuté ligne par ligne
  • Une erreur peut bloquer l'exécution du reste du script
  • Exemple d'erreur fréquente : élément HTML non trouvé 
const btn = document.getElementById("bouton");
btn.addEventListener("click", () => {
  console.log("Clique");
});
// Erreur si btn est null — sécuriser avec :
btn?.addEventListener("click", () => {
  console.log("Clique");
});

Le chaînage optionnel (?.) évite l’erreur si getElementById renvoie null.

Console et outils développeur (pratique)

  • Ouvrir la page dans le navigateur
  • Clic droit → Inspecter
  • Onglet Console
  • Voir les messages, erreurs et tests
console.log("Page chargée");
console.error("Erreur détectée");
console.warn("Attention");
debugger; // pause l’exécution si les DevTools sont ouverts

La console est votre meilleur ami pour comprendre et corriger votre code ; placez un point d’arrêt dans l’onglet Sources ou utilisez debugger.

4. Bases et syntaxe JavaScript

  • La syntaxe définit les règles d'écriture du langage JavaScript.
  • Une petite erreur de syntaxe peut bloquer tout le programme.
  • Comprendre les bases évite les erreurs fréquentes chez les débutants.

Instructions et expressions

Différence claire

  • Instruction : ordre donné à JavaScript pour exécuter une action.
  • Expression : produit une valeur. 
// Instruction
let x = 5;

// Expression
5 + 3      // produit 8

// Expression dans une instruction
let y = 5 + 3;

Commentaires et sensibilité à la casse

Commentaires

  • Les commentaires servent à expliquer le code.
  • Ils ne sont pas exécutés par JavaScript. 
// Commentaire sur une ligne

/*
  Commentaire
  sur plusieurs lignes
*/

Sensibilité à la casse

JavaScript fait la différence entre majuscules et minuscules. 

let nom = "Ali";
let Nom = "Sara";

console.log(nom); // "Ali"
console.log(Nom); // "Sara"

Point-virgule en JavaScript

  • Le point-virgule ; termine une instruction.
  • JavaScript peut l'ajouter automatiquement (ASI).
  • Mais dans certains cas, il est obligatoire. 
// Correct
let a = 10;
let b = 20;

// Problème possible sans ;
let x = 10
(function () {
  console.log("Hello");
})();

Bonne pratique : toujours utiliser le point-virgule.

Indentation et lisibilité du code

L'indentation rend le code lisible et compréhensible.

Mauvaise indentation 

if(true){
console.log("OK");
if(false){
console.log("Non");
}
}

Bonne indentation 

if (true) {
  console.log("OK");
  if (false) {
    console.log("Non");
  }
}

Un code lisible est plus facile à corriger et à maintenir.

5. Variables – Les boîtes de rangement

  • Une variable est un espace en mémoire pour stocker une valeur.
  • Analogie : une boîte avec une étiquette dessus.
  • L'étiquette est le nom de la variable, le contenu est la valeur. 
// Boîte nommée age contenant la valeur 25
let age = 25;

Déclarer une variable

JavaScript propose trois mots-clés pour déclarer des variables. 

const nom = "Ali";
let score = 10;
var ancien = true;
  • const : valeur qui ne change pas
  • let : valeur qui peut changer
  • var : ancienne méthode (à éviter)

const vs let vs var

Mot-clé Peut changer ? Portée À utiliser ?
const ❌ Non Bloc ✅ Oui (par défaut)
let ✅ Oui Bloc ✅ Oui
var ✅ Oui Fonction ❌ À éviter

Portée (scope) expliquée simplement

La portée indique où une variable est accessible. 

if (true) {
  let x = 10;
  const y = 20;
}

// x et y n'existent plus ici
if (true) {
  var z = 30;
}

console.log(z); // 30 (problème)

Hoisting (explication simple)

  • JavaScript "déplace" les déclarations vers le haut du fichier.
  • var est hoisté et initialisé à undefined.
  • let et const sont hoistés mais non accessibles avant déclaration. 
console.log(a); // undefined
var a = 10;
console.log(b); // Erreur
let b = 20;

Conventions de nommage

camelCase (recommandé)

  • Commence par une minuscule
  • Chaque nouveau mot commence par une majuscule
  • Pas d'espaces, pas de caractères spéciaux
// Correct
let userName = "Ali";
let totalPrice = 150;

// Incorrect
let user_name = "Ali";
let User Name = "Ali";

6. Types de données primitifs

  • Un type de donnée définit la nature d'une valeur.
  • JavaScript possède plusieurs types primitifs.
  • Les types primitifs stockent une valeur simple.

Number (nombres)

Le type Number représente les nombres entiers et décimaux. 

let age = 25;
let price = 19.99;
let temperature = -5;
  • Pas de distinction entre entier et décimal
  • Un seul type pour tous les nombres

String (texte)

Le type String permet de stocker du texte. 

let firstName = "Ali";
let lastName = 'Zaroual';
let message = `Bonjour ${firstName}`;
  • Guillemets simples, doubles ou backticks
  • Les backticks permettent l'interpolation

Boolean (vrai / faux)

Le type Boolean contient seulement deux valeurs. 

let isLoggedIn = true;
let isAdmin = false;
  • Très utilisé dans les conditions
  • Résultat des comparaisons

null et undefined (différence importante)

  • undefined : variable déclarée mais sans valeur
  • null : absence de valeur volontaire 
let x;
console.log(x); // undefined

let y = null;
console.log(y); // null

null = "je sais qu'il n'y a rien"

Symbol et typeof

Symbol (mention rapide)

Symbol crée des valeurs uniques. Utilisé dans des cas avancés. 

const id = Symbol("id");

Opérateur typeof 

typeof 10        // "number"
typeof "hello"   // "string"
typeof true      // "boolean"
typeof undefined // "undefined"
typeof null      // "object" (erreur historique)

Typage dynamique

JavaScript est un langage à typage dynamique. 

let value = 10;
value = "dix";
value = true;

Avantage

Flexible et rapide à écrire

Inconvénient

Erreurs possibles si on ne fait pas attention

7. Chaînes de caractères (Strings)

  • Une chaîne de caractères représente du texte.
  • Très utilisée pour afficher des messages, noms, phrases.
  • JavaScript propose plusieurs façons de créer des strings.

Créer des chaînes de caractères

Une chaîne peut être écrite avec différents types de guillemets. 

let a = "Bonjour";
let b = 'Salut';
let c = `Hello`;
  • Guillemets simples et doubles: classiques
  • Backticks (` `): méthode moderne

Template literals (backticks)

Les template literals permettent d'insérer des variables facilement. 

let name = "Ali";
let age = 25;

let message = `Bonjour ${name}, tu as ${age} ans`;
  • Plus lisible
  • Recommandé aujourd'hui

Concaténation des chaînes

Ancienne méthode

let msg = "Bonjour " + name + ", âge: " + age;

Peu lisible et source d'erreurs

Méthode moderne

let msg = `Bonjour ${name}, âge: ${age}`;

Lisible et propre

Méthodes utiles sur les chaînes 

let text = "  JavaScript  ";

text.length;              // 14
text.toUpperCase();       // "  JAVASCRIPT  "
text.toLowerCase();       // "  javascript  "
text.trim();              // "JavaScript"
text.includes("Script");  // true
"abc".at(-1);             // "c" (dernier caractère, ES2022)
"hello".slice(1, 4);      // "ell"
"5".padStart(3, "0");     // "005"
"x".repeat(3);            // "xxx"
  • .length : longueur de la chaîne
  • .trim() / .trimStart() / .trimEnd() : espaces
  • .includes(), .startsWith(), .endsWith() : recherche
  • .slice() : sous-chaîne sans modifier l’original
  • .charAt() / .at() : caractère à l’index

Échapper les caractères spéciaux

Pour utiliser certains caractères, il faut les échapper avec \. 

let quote = "Il a dit : \"Bonjour\"";
let path = "C:\\Users\\Ali";
  • \" pour les guillemets
  • \\ pour le backslash
  • \n pour une nouvelle ligne

Erreurs courantes avec les guillemets

Incorrect 

let msg = "Bonjour';
let text = 'Il a dit "Salut"';

Correct 

let msg = "Bonjour";
let text = 'Il a dit "Salut"';
let text2 = `Il a dit "Salut"`;

Toujours ouvrir et fermer les guillemets correctement.

8. Nombres et Math

  • En JavaScript, tous les nombres ont le même type : number.
  • Il n'y a pas de différence entre entiers et décimaux.
  • Les calculs sont faciles, mais il faut connaître les pièges.

Les nombres en JavaScript 

let a = 10;       // entier
let b = 3.14;     // décimal
let c = -5;       // négatif
let d = 1e3;      // 1000 (notation scientifique)
  • Un seul type : number
  • Basé sur le standard IEEE 754

Opérations mathématiques de base 

10 + 5   // addition → 15
10 - 5   // soustraction → 5
10 * 5   // multiplication → 50
10 / 5   // division → 2
10 % 3   // modulo → 1
2 ** 3   // puissance → 8
  • % : reste de la division
  • ** : puissance (moderne)

L'objet Math 

Math.round(4.6);   // 5
Math.floor(4.6);   // 4
Math.ceil(4.1);    // 5
Math.trunc(4.9);   // 4 (partie entière)
Math.abs(-3);      // 3
Math.max(1, 5, 2); // 5
Math.min(1, 5, 2); // 1
Math.sqrt(16);     // 4
Math.pow(2, 3);    // 8
Math.random();     // [0, 1[

// Entier aléatoire entre min et max inclus
const rnd = (min, max) =>
  Math.floor(Math.random() * (max - min + 1)) + min;
  • round / floor / ceil / trunc : arrondis différents
  • max / min : comparer plusieurs valeurs
  • random : toujours combiner avec floor pour un entier

parseInt et parseFloat

Utilisés pour convertir du texte en nombre. 

parseInt("42");        // 42
parseInt("42px");      // 42
parseInt("ff", 16);    // 255 (base 16)
parseFloat("3.14");    // 3.14
parseFloat("3.14kg");  // 3.14

Number("42");          // 42 — stricte (échoue si texte invalide)
+"42";                 // 42 (unaire +)

Préférer Number() ou parseFloat selon le cas ; toujours vérifier Number.isNaN() après conversion utilisateur.

Problèmes de précision 

0.1 + 0.2 === 0.3; // false
0.1 + 0.2;         // 0.30000000000000004
  • Problème lié au stockage binaire
  • Commun à beaucoup de langages
// Solution courante
Number((0.1 + 0.2).toFixed(2)); // 0.3

NaN et Infinity 

NaN;            // Not a Number
10 / "a";       // NaN
Infinity;       // Infini
10 / 0;         // Infinity
  • NaN : résultat invalide
  • Infinity : division par zéro
isNaN("abc");   // true

Erreurs courantes de conversion

Problème 

"10" + 5;   // "105"
"10" - 5;   // 5

Solution 

Number("10") + 5; // 15
parseInt("10") + 5;

Attention aux calculs avec des chaînes de caractères.

9. Booléens et comparaisons

Un booléen représente une valeur logique. 

true
false
  • Résultat des comparaisons
  • Utilisé dans les conditions

Opérateurs de comparaison 

10 > 5     // true
10 < 5     // false
10 >= 10  // true
10 <= 9   // false

5 == "5"   // true (conversion)
5 === "5"  // false (STRICT)

5 != "5"   // false
5 !== "5"  // true

== vs === (CRUCIAL)

== (à éviter)

0 == false   // true
"" == false  // true
null == undefined // true

=== (à utiliser)

0 === false  // false
"" === false // false
null === undefined // false

Règle d'or: toujours utiliser === et !==

Opérateurs logiques 

true && false  // false
true || false  // true
!true          // false
  • && : ET — retourne la première valeur falsy ou la dernière truthy.
  • || : OU — retourne la première truthy.
  • ! : NON
  • ?? (nullish coalescing) : a ?? b renvoie b seulement si a est null ou undefined (pas pour 0 ou ""). 
const port = options.port ?? 3000;
const name = input || "invité";

Valeurs truthy et falsy

En condition, certaines valeurs sont considérées comme false. 

// FALSY
false
0
""
null
undefined
NaN
// TRUTHY
"0"
[]
{}
42

Attention : || traite 0 et "" comme falsy ; ?? ne remplace que null / undefined.

Pièges courants 

if ("0") {
  console.log("Exécuté"); // OUI
}

if ([]) {
  console.log("Exécuté"); // OUI
}

11. Boucles – La répétition

  • Les boucles permettent d’exécuter un bloc plusieurs fois sans dupliquer le code.
  • On choisit la boucle selon le cas : nombre d’itérations connu (for), condition (while), parcours de tableau (for...of, forEach) ou clés d’objet (for...in avec prudence).
  • Les méthodes de tableau (map, filter, etc.) remplacent souvent une boucle classique pour plus de clarté.

Boucle for classique 

for (let i = 0; i < 5; i++) {
  console.log(i);
}

// Boucles auxiliaires : break / continue
for (let i = 0; i < arr.length; i++) {
  console.log(arr[i]);
}

Les trois parties du for sont optionnelles ; pour les tableaux, for...of ou .forEach sont souvent plus lisibles.

while et do...while 

let i = 0;
while (i < 3) {
  console.log(i);
  i++;
}
let j = 0;
do {
  console.log(j);
  j++;
} while (j < 1);

for...of et for...in

for...of : valeurs d’un itérable (tableau, chaîne, Map, Set…). for...in : clés énumérables (surtout sur objets ; inclut parfois le prototype — utiliser hasOwn ou filtrer). 

for (const n of [1, 2, 3]) {
  console.log(n);
}

for (const ch of "abc") {
  console.log(ch); // a, b, c
}

for (const key in user) {
  if (Object.hasOwn(user, key)) {
    console.log(key, user[key]);
  }
}

Pour les tableaux, forEach (sans break) ou for...of avec break/continue selon le besoin.

break et continue 

for (let i = 0; i < 5; i++) {
  if (i === 2) continue; // passe à l’itération suivante
  if (i === 4) break;    // sort de la boucle
  console.log(i);
}

continue et break s’appliquent aussi à while et for...of ; dans forEach, utiliser return pour « continuer » (pas d’équivalent direct à break).

Erreurs courantes 

// Boucle infinie
while (true) {}

// Off-by-one
for (let i = 0; i <= 5; i++) {}

11. Fonctions – Les outils réutilisables

  • Une fonction est un bloc de code réutilisable.
  • Elle permet d'éviter la répétition et de structurer le programme.
  • Une fonction peut recevoir des données et retourner un résultat.

Déclaration de fonction

La forme classique pour définir une fonction. 

function addition(a, b) {
  return a + b;
}

let resultat = addition(3, 4); // 7
  • Lisible et claire
  • Hoistée (connue avant son appel)

Expression de fonction

Une fonction stockée dans une variable. 

const multiplication = function(a, b) {
  return a * b;
};

multiplication(2, 3); // 6
  • Non hoistée
  • Très utilisée avec les callbacks

Fonctions fléchées (MODERNE)

Syntaxe courte et moderne (ES6). Recommandée aujourd'hui. 

// Forme classique
const carre = function(n) {
  return n * n;
};

// Fonction fléchée
const carre = n => n * n;

// Plusieurs paramètres ou corps multi-lignes
const add = (a, b) => a + b;
const log = (msg) => {
  console.log(msg);
};
  • Syntaxe courte ; idéale pour callbacks (map, événements…).
  • Pas de this propre : this vient du contexte parent (utile dans les classes / objets avec précaution).
  • Pas d’objet arguments ; utiliser les paramètres reste ...args si besoin.

Paramètres et arguments

Les paramètres sont les variables de la fonction. 

function saluer(nom) {
  console.log("Bonjour " + nom);
}

saluer("Ali"); // "Ali" est un argument

Valeurs par défaut des paramètres 

function calculPrix(prix, taxe = 0.2) {
  return prix + prix * taxe;
}

calculPrix(100);    // 120
calculPrix(100, 0); // 100

Évite les erreurs si un argument est manquant.

return (retourner une valeur) 

function addition(a, b) {
  return a + b;
}

let total = addition(2, 3);
  • return renvoie une valeur
  • Arrête l'exécution de la fonction

Fonctions sans return 

function afficherMessage() {
  console.log("Hello");
}

let result = afficherMessage(); // undefined

Sans return, la fonction retourne undefined.

Callback functions (introduction)

Une callback est une fonction passée en argument. 

function traiter(valeur, action) {
  action(valeur);
}

traiter(5, n => console.log(n * 2));

Fonctions pures (bonne pratique)

Une fonction pure :

  • Dépend seulement de ses paramètres
  • Ne modifie pas l'extérieur
// Fonction pure
const somme = (a, b) => a + b;

// Impure
let total = 0;
function ajouter(n) {
  total += n;
}

Erreurs courantes avec les fonctions 

// Oublier return
function test() {
  5 + 5;
}

// Appeler la fonction trop tôt
const f = afficher(); // ❌
const f = afficher;  // ✅

Bien distinguer appeler une fonction et la passer.

Tableaux (Arrays) – Les listes

  • Un tableau est une liste ordonnée de valeurs.
  • Il peut contenir différents types de données.
  • Les indices sont numériques ; la longueur est donnée par .length.
let numbers = [1, 2, 3, 4];
let copy = [...numbers];        // copie superficielle (spread)
let fromStr = Array.from("ab"); // ["a","b"]
let filled = new Array(3).fill(0); // [0,0,0]

Array.from accepte aussi une longueur + fonction de mapping pour générer des lignes (ex. grilles).

Créer un tableau et accéder aux éléments 

let fruits = ["Pomme", "Banane", "Orange"];

console.log(fruits[0]); // Pomme
console.log(fruits[1]); // Banane
  • L'index commence à 0
  • Accès avec les crochets []

Propriété .length et méthodes de base 

let arr = [1, 2, 3];

arr.length; // 3

arr.push(4);   // ajoute à la fin
arr.pop();     // supprime le dernier
arr.shift();   // supprime le premier
arr.unshift(0); // ajoute au début

let merged = arr.concat([5, 6]); // nouveau tableau
// Souvent : [...arr, 5, 6]
  • push/pop : fin du tableau ; shift/unshift : début (coûteux sur grands tableaux).
  • concat ou spread pour fusionner sans muter.
  • reverse / sort modifient sur place ; préférer toSorted / toReversed (copie) si disponible.

12. Fonctions d’ordre supérieur (High Order Functions)

  • Une fonction d’ordre supérieur reçoit une fonction en paramètre.
  • Les méthodes modernes des tableaux utilisent des callbacks.
  • Ces méthodes ne modifient généralement pas le tableau original.
array.method((value, index, array) => {
  // logique
});

value = élément courant | index = position | array = tableau original

.map() – Transformer les éléments 

let numbers = [10, 20, 30];

let doubled = numbers.map((value, index, array) => {
  console.log(value, index);
  return value * 2;
});

console.log(doubled); // [20, 40, 60]
  • Retourne un nouveau tableau
  • Doit retourner une valeur
  • Ne modifie pas l’original

.filter() – Filtrer selon une condition 

let numbers = [10, 20, 30];

let result = numbers.filter((value, index, array) => {
  return value > 15;
});

console.log(result); // [20, 30]
  • Le callback doit retourner true ou false
  • true → élément conservé
  • false → élément supprimé

.find(), .findIndex() et .forEach() 

let numbers = [10, 20, 30];

let found = numbers.find((value) => value === 20);
let idx = numbers.findIndex((value) => value === 20);

numbers.forEach((value, index) => {
  console.log(value);
});
  • find : premier élément qui valide le test, ou undefined.
  • findIndex : index de cet élément, ou -1.
  • forEach : parcourt tout (pas de return utile pour sortir tôt — utiliser une boucle classique si besoin).

.reduce() – Combiner les valeurs 

let numbers = [10, 20, 30];

let total = numbers.reduce(
  (accumulator, currentValue, index, array) => {
    return accumulator + currentValue;
  },
  0
);

console.log(total); // 60
  • accumulator → valeur accumulée
  • currentValue → élément courant
  • 0 → valeur initiale

Mutation vs Immutabilité 

let arr = [1, 2, 3];

// Mutation ❌
arr.push(4);

// Immutabilité ✅
let newArr = [...arr, 4];
  • Mutation = modifier le tableau original
  • Immutabilité = créer une copie
  • Fondamental pour React

Spread operator et immutabilité 

let arr = [1, 2, 3];
let newArr = [...arr, 4];

let obj = { a: 1 };
let clone = { ...obj, b: 2 };
  • Le spread ... « copie » les éléments ou propriétés dans un nouveau littéral.
  • Copie superficielle : objets imbriqués restent partagés.
  • Très utilisé avec React pour mettre à jour l’état sans muter.

Erreurs courantes avec les index 

let arr = [1, 2, 3];

console.log(arr[5]); // undefined
  • Accès hors limite → undefined
  • Attention aux boucles mal définies

13. Objets – Les structures clé-valeur

  • Un objet contient des paires clé-valeur.
  • Permet de représenter des entités réelles (utilisateur, produit, config…).
  • Méthodes utiles : Object.keys, Object.values, Object.entries pour parcourir ; structuredClone(obj) pour une copie profonde (navigateurs modernes). 
let user = {
  name: "Ali",
  age: 25
};

Notation point vs crochet 

user.name;      // notation point
user["age"];    // notation crochet
  • Point: simple et lisible
  • Crochet: utile pour clé dynamique

Ajouter, modifier, supprimer 

user.email = "ali@mail.com";
user.age = 30;
delete user.email;

Méthodes et mot-clé this 

let user = {
  name: "Ali",
  greet() {
    console.log("Bonjour " + this.name);
  }
};

user.greet();

this fait référence à l'objet courant.

Spread et destructuration

Le spread (...) « déplie » un itérable ; la destructuration extrait des propriétés ou des éléments en variables. 

// Objet : copie superficielle + nouvelles clés
let newUser = { ...user, city: "Casablanca" };

// Tableau : concaténer ou copier
let a = [1, 2];
let b = [...a, 3, 4];

// Destructuration objet
let { name, age } = user;
let { name: nom, age: ans } = user; // renommage

// Valeurs par défaut + reste
let { role = "user", ...rest } = user;

// Destructuration tableau
let [first, second, ...others] = [10, 20, 30, 40];

Le spread sur objets/tableaux est une copie superficielle (les objets imbriqués restent référencés).

JSON.stringify et JSON.parse

JSON est le format texte le plus utilisé pour échanger des données avec des API ; seuls types supportés : objets, tableaux, chaînes, nombres, booléens et null (pas de Date, undefined ni fonctions). 

let json = JSON.stringify(user);
let pretty = JSON.stringify(user, null, 2); // indentation lisible

// Filtrer les clés (ex. exclure un mot de passe)
let safe = JSON.stringify(user, ["name", "age"]);

let obj = JSON.parse(json);

// Reviver : recréer des types (ex. Date) à la lecture
let data = JSON.parse(json, (key, value) =>
  key === "createdAt" ? new Date(value) : value
);
  • stringify : objet → chaîne ; 2ᵉ argument = remplacement des clés.
  • parse : chaîne → objet ; 2ᵉ argument optionnel = fonction de revival.
  • Les clés dont la valeur est undefined sont omises par stringify.

14. Méthodes les plus utilisées (Essentiel développeur)

  • Les méthodes permettent de manipuler facilement les données.
  • Une grande partie du JavaScript quotidien repose sur ces patterns.
  • Très utilisées en React, APIs, Node.js et manipulation de fichiers JSON.
  • À connaître côté tableaux : map, filter, find, findIndex, some, every, reduce, includes, slice / splice, flat / flatMap.

Maîtriser ces méthodes = base solide pour lire et écrire du code pro et des revues de code en équipe.

Arrays : includes() et indexOf() 

let fruits = ["Pomme", "Banane", "Orange"];

fruits.includes("Banane"); // true
fruits.indexOf("Orange");  // 2
  • includes() → vérifie la présence (avec ===).
  • indexOf() / lastIndexOf() → index ou -1.
  • findIndex() → comme find mais renvoie l’index (ou -1).
  • at() → accès par index (ex. -1 = dernier élément, ES2022).

slice() vs splice() 

let arr = [1, 2, 3, 4];

arr.slice(1, 3);   // [2, 3] — extrait sans modifier arr

// splice(start, deleteCount, ...itemsÀInsérer)
arr.splice(1, 2);       // supprime 2 éléments à partir de l’index 1
arr.splice(1, 0, 99);   // insère 99 sans supprimer
  • slice() → copie superficielle d’un morceau ; ne modifie pas l’original.
  • splice() → modifie sur place (suppression / insertion) et renvoie les éléments retirés.
  • concat() → fusionne sans muter (souvent remplacé par le spread [...a, ...b]).

some() et every() 

let numbers = [10,20,30];

numbers.some(n => n > 25);  // true
numbers.every(n => n > 5);  // true
  • some() → au moins un élément valide
  • every() → tous les éléments valides
Tableaux

flat(), flatMap() — tableaux imbriqués

flat aplatit les niveaux ; flatMap combine map + flat(1) en une passe (très utilisé en données).

flat — profondeur

const nested = [1, [2, [3, 4]]];
nested.flat();    // [1, 2, [3, 4]]
nested.flat(2);   // [1, 2, 3, 4]

flatMap — un niveau

const phrases = ["a b", "c d"];
const lettres = phrases.flatMap((p) =>
  p.split(" ")
);
// ["a","b","c","d"]

À retenir : flat(Infinity) peut aplatir profondément mais coûte cher sur très gros tableaux — préférer une profondeur connue.

Tableaux

Tri, copie immuable & utilitaires 

const n = [3, 1, 20, 2];
[...n].sort((a, b) => a - b);   // copie triée (nombres)
n.toSorted((a, b) => a - b);     // idem sans muter n (ES2023)

const items = ["a", "b", "c"];
items.join(" · ");               // "a · b · c"
Array.isArray(items);            // true

const copy = items.toReversed(); // copie inversée (ES2023)
items.with(1, "B");              // nouveau tab. : index 1 → "B"
Note : sort() modifie le tableau sur place et convertit en chaîne par défaut — pour des nombres, toujours un comparateur (a,b) => a-b. Préférer toSorted si vous voulez garder l’original intact.
Chaînes

replace() et replaceAll()

// "Bonjour Sara Ali"

text.replaceAll("Ali", "Sara");
// "Bonjour Sara Sara"

// replace avec fonction ou regex globale
text.replace(/Ali/g, "Sara");

Compléter une chaîne

"42".padStart(5, "0");  // "00042"
"hi".padEnd(6, ".");     // "hi...."

Répéter & espaces

"-".repeat(10);
"  x  ".trim();

replace ne remplace que la première occurrence sauf regex avec flag g ou replaceAll (ES2021).

split() et join() 

let sentence = "JavaScript est puissant";

let words = sentence.split(" ");
console.log(words);

words.join("-");
  • split() → string → tableau
  • join() → tableau → string

Astuce : "a,b,c".split(",") → ["a","b","c"] ; "hello".split("") → tableau de caractères.

Chaînes

Sous-chaînes, recherche & comparaison

slice / substring

s.slice(-6);        // "script"
s.substring(0, 4);  // "Java"

indexOf / match

// Ordre alphabétique localisé (tri, recherche)

startsWith() et endsWith()

Très utilisé pour validation de formulaires.

  name: "Ali",

Object.assign()

console.log(updated);

Fusionne des propriétés dans la cible (copie superficielle). Souvent remplacé par le spread : { ...user, age: 30 }.

Vérifie si une propriété existe.

17. Dates en JavaScript

  • Les dates permettent de gérer le temps dans une application.
  • Utilisées pour événements, réservations, authentification, logs.
  • JavaScript utilise l’objet Date.
let today = new Date();

Une date représente un moment précis dans le temps.

Création d’une date 

// Date actuelle
new Date();

// Depuis une chaîne (ISO recommandé)
new Date("2026-03-10");

// Année, mois, jour (heure locale)
new Date(2026, 2, 10);

// Depuis un timestamp (ms depuis 1970)
new Date(1710000000000);
  • ⚠ Les mois commencent à 0 dans le constructeur (année, mois, jour)
  • 0 = Janvier, 11 = Décembre
  • Les chaînes non ISO peuvent être interprétées différemment selon le navigateur.

Date actuelle 

let now = new Date();

console.log(now);
  • Retourne la date et l’heure actuelles.
  • Dépend du fuseau horaire utilisateur.

Lire une date 

let d = new Date();

d.getFullYear();
d.getMonth();    // 0–11
d.getDate();     // jour du mois
d.getDay();      // 0 = dimanche … 6 = samedi

d.getTime();     // ms depuis le 1er janv. 1970 (UTC)
// Variantes UTC : getUTCFullYear(), getUTCMonth(), …
  • getMonth() → mois (0-11)
  • getDay() → jour de la semaine (0 = dimanche), pas confondre avec getDate()
  • getTime() → timestamp (pratique pour comparer ou stocker)

Heures et minutes 

let d = new Date();

d.getHours();
d.getMinutes();
d.getSeconds();
d.getMilliseconds();

// UTC (serveurs, logs)
d.getUTCHours();
d.getUTCMinutes();

Permet horaires, timers et affichage temps réel ; utiliser UTC pour des données partagées entre fuseaux (stockage backend).

Modifier une date 

let d = new Date();

d.setFullYear(2030);
d.setMonth(5);
d.setDate(20);
  • Les méthodes set modifient la date originale.

Formatage des dates 

let d = new Date();

d.toLocaleDateString();
d.toLocaleTimeString();
d.toLocaleString("fr-FR");

// Options fines (jour, mois long, fuseau…)
d.toLocaleString("fr-FR", {
  weekday: "long",
  year: "numeric",
  month: "long",
  day: "numeric",
});

// API moderne (internationalisation)
new Intl.DateTimeFormat("fr-FR", {
  dateStyle: "medium",
  timeStyle: "short",
}).format(d);
  • Format automatique selon la locale (langue / région).
  • Intl.DateTimeFormat : contrôle fin pour apps multilingues.

Timestamp et Date.now() 

Date.now();           // équivalent rapide à new Date().getTime()

let d = new Date();
d.valueOf();          // comme getTime()
d.getTime();          // ms depuis epoch (UTC)

Le nombre de millisecondes depuis le 1er janvier 1970 UTC (epoch) sert aux durées et aux comparaisons. 

let timestamp = Date.now();
let date = new Date(timestamp);

Calcul entre deux dates 

let start = new Date("2026-03-01");
let end = new Date("2026-03-10");

let diff = end - start;

let days = diff / (1000 * 60 * 60 * 24);

console.log(days);

Conversion millisecondes → jours.

Fuseaux horaires (Bonnes pratiques)

  • Toujours stocker les dates en UTC.
  • Afficher selon la localisation utilisateur.
  • Utiliser ISO format : YYYY-MM-DD.
new Date().toISOString();

Erreurs courantes avec les dates 

// Mois commence à 0
new Date(2026, 0, 1);

// Mauvais format
new Date("10/03/2026");
  • Confusion mois indexés à 0
  • Formats différents selon navigateurs
  • Fuseaux horaires ignorés

Toujours préférer le format ISO.

Dates

Comparer, trier & « date invalide »

Comparer deux moments

const a = new Date("2026-03-01");
const b = new Date("2026-03-10");
a < b;                 // true
a.getTime() === b.getTime();

Trier un tableau de dates

const dates = [
  new Date("2026-01-15"),
  new Date("2025-12-01"),
];
dates.sort((x, y) => x - y);
const bad = new Date("pas une date");
isNaN(bad.getTime()); // true → toujours vérifier après parse

Les opérateurs < > sur Date appellent valueOf() (timestamp). Pour l’égalité stricte, utiliser getTime().

Dates

ISO string, UTC affiché & durée lisible 

const d = new Date();
d.toISOString();
// "2026-03-24T14:30:00.000Z" — toujours en UTC (Z)

// Afficher côté utilisateur (fuseau local)
d.toLocaleString("fr-FR", {
  timeZone: "Africa/Casablanca",
  dateStyle: "full",
  timeStyle: "short",
});

// Durée entre deux dates (ex. minute)
const ms = fin - debut;
const minutes = Math.floor(ms / 60000);
Bonnes pratiques : stocker en base en UTC (toISOString) ; formater à l’affichage avec la locale et le fuseau de l’utilisateur. Pour les durées précises (rappels, timers), tout reste en millisecondes.
Module — DOM

17. Le DOM — Manipuler la page web

  • Le DOM permet à JavaScript de lire et modifier la page après chargement.
  • Chaque balise est un objet ; les méthodes sont sur document et sur chaque élément.
// Exemple minimal : tout passe par document ou une référence d’élément
const titre = document.querySelector("h1");
titre.textContent = "Bonjour";

document.body.classList.add("theme-dark");

DOM = pont entre JavaScript et le HTML rendu dans le navigateur.

Qu'est-ce que le DOM ?

Le navigateur transforme le HTML en un arbre appelé DOM (Document Object Model). 

<body>
  <h1>Titre</h1>
  <p>Texte</p>
</body>

En JS, le même arbre est navigable :

// document est le point d’entrée
console.log(document.body.children.length); // 2 <h1> et <p>

const p = document.querySelector("p");
console.log(p.childNodes); // nœuds Text à l’intérieur du <p>
  • Chaque balise → souvent un Element ; le texte → Text.
  • document est une instance de Document (nœud racine logique).

Visualiser le DOM (DevTools)

  • Clic droit → Inspecter ; onglet Elements = arbre vivant.
  • Console : commandes utiles une fois un nœud sélectionné dans Elements.
// $0 = dernier élément sélectionné dans l’onglet Elements
$0;
$0.textContent;
$0.getAttribute("class");

// Rechercher dans la page depuis la console
document.querySelector("nav a.active");

// Surveiller le rendu (layout)
console.log($0.getBoundingClientRect());

Astuce : copy($0.outerHTML) copie le HTML de l’élément dans le presse-papiers.

Sélection — méthodes « classiques » 

// Un seul élément par id (sans #) — null si absent
const root = document.getElementById("app");

// Plusieurs classes possibles dans une chaîne
const cards = document.getElementsByClassName("card highlight");

// Toutes les <li> du document
const items = document.getElementsByTagName("li");

// Radio / checkbox / champs avec name="newsletter"
const fields = document.getElementsByName("newsletter");

// Parcourir une HTMLCollection (for...of ou index)
for (const li of items) {
  console.log(li.textContent);
}
// Copie tableau si besoin d’array (filter, map…)
const arr = Array.from(document.getElementsByTagName("p"));
  • HTMLCollection (souvent live) : si vous ajoutez un <li>, items.length peut augmenter.
  • getElementById : le plus direct pour un id ; pas de préfixe #.

Sélection — querySelector & alliés 

// Sélecteurs CSS complets (pseudo-classes, attributs…)
document.querySelector("#sidebar .item:first-child");
document.querySelector('input[type="checkbox"]:checked');

// Tous les matches — NodeList (souvent statique)
const nodes = document.querySelectorAll(".card");
nodes.forEach((el) => el.classList.add("seen"));

// :scope = contexte (ex. enfants directs d’un conteneur)
container.querySelectorAll(":scope > li");

// Tests sur l’élément déjà récupéré
if (btn.matches(".btn-primary")) { /* … */ }
const form = el.closest("form"); // remonte les parents jusqu’à un <form>

document.documentElement.lang; // <html lang>
document.body;

querySelector renvoie null si rien ; prévoir if ou ?.

Types de nœuds (rappel)

  • Document — document ; Element — balises ; Text / Comment — contenu.
  • DocumentFragment — parent temporaire, pas dans le DOM jusqu’à insertion.
function logNode(node) {
  if (node.nodeType === Node.ELEMENT_NODE) {
    console.log("Élément :", node.nodeName);
  } else if (node.nodeType === Node.TEXT_NODE) {
    console.log("Texte :", node.nodeValue);
  }
}

// Fragment : construire hors écran puis injecter un seul reflow
const frag = document.createDocumentFragment();
for (let i = 0; i < 100; i++) {
  const li = document.createElement("li");
  li.textContent = `Ligne ${i}`;
  frag.appendChild(li);
}
document.querySelector("ul").appendChild(frag); // une seule insertion

Navigation dans l’arbre 

// Exemple : <ul><li id="x">a</li><li>b</li></ul>
const li = document.getElementById("x");
const ul = li.parentElement;

for (const child of ul.children) {
  console.log(child.tagName); // LI, LI
}

li.nextElementSibling.textContent; // "b"
li.previousElementSibling;         // null

// Nœuds « tout type » (espaces = nœuds Text !)
console.log(ul.childNodes.length); // peut être > children.length
ul.firstChild;  // souvent un nœud #text si le HTML est indenté

Règle : parcourir des balises → *Element* ; analyser le texte brut ou les commentaires → *Node*.

Modifier le contenu 

const h = document.querySelector("h1");

// Texte brut — rapide, pas d’interprétation HTML
h.textContent = `Utilisateur : ${name}`;

// Comme le texte affiché à l’écran (sauts de ligne, style display:none ignoré…)
banner.innerText = "Ligne 1\nLigne 2";

// Parse le HTML (insérer du markup)
box.innerHTML = "<p>Paragraphe</p>\n<ul><li>OK</li></ul>";

// Remplace la balise elle-même (l’élément h n’existe plus après)
// h.outerHTML = "<section></section>";

// Insérer du HTML à un endroit précis sans écraser tout le contenu
el.insertAdjacentHTML("beforeend", "<span>+</span>");
// positions : "beforebegin" | "afterbegin" | "beforeend" | "afterend"
  • textContent pour données utilisateur ; innerHTML seulement si HTML maîtrisé ou sanitisé.

Attributs, propriétés et data-* 

const link = document.querySelector("a.external");

// Lire / écrire le HTML tel qu’affiché dans le source

link.setAttribute("target", "_blank");
link.setAttribute("rel", "noopener noreferrer");
link.toggleAttribute("disabled"); // ajoute ou enlève l’attribut booléen

// data-* <div data-user-id="42" data-role="admin">
const id = div.dataset.userId;   // "42" (string)
div.dataset.role = "guest";    // data-role="guest"

// Champs : propriétés souvent plus pratiques que getAttribute

checkbox.checked = true;
input.valueAsNumber; // pour type="number"

getAttribute("value") peut différer de input.value après saisie utilisateur — pour l’état courant, utiliser les propriétés.

Styles inline et classes 

const box = document.querySelector(".box");

box.style.backgroundColor = "blue";
box.style.setProperty("--gap", "1rem");

// Style calculé après CSS (lecture)
getComputedStyle(box).color;

// Classes — préféré aux styles inline répétés
box.classList.add("active");
box.classList.toggle("hidden", condition);
box.classList.replace("old", "new");

Les propriétés CSS en JS sont en camelCase ; classList évite de concaténer des chaînes de classes à la main.

Créer, insérer et supprimer des éléments 

const li = document.createElement("li");
li.textContent = "Nouvel élément";

const ul = document.querySelector("ul");
ul.append(li);                    // fin + accepte plusieurs args
ul.prepend(li);                   // début
ul.append("texte", li, autreLi);  // string ou nœud

const ref = ul.querySelector("li:first-child");
ref.before(li);  // juste avant ref
ref.after(li);   // juste après

ref.remove();               // supprime ref du DOM
ref.replaceWith(newSpan);   // remplace ref par un autre nœud

// Ancienne API (toujours valide)
ul.appendChild(li);
ul.insertBefore(li, ref);

Événements — écouter et options 

function onClick(event) {
  // Délégation : clic sur un bouton dans une liste
  const btn = event.target.closest("button[data-id]");
  if (!btn) return;

  event.preventDefault();     // annule l’action par défaut (lien, form…)
  event.stopPropagation();    // n’envoie pas l’événement plus haut

  console.log(event.currentTarget); // élément qui a addEventListener (souvent le ul)
}

document.getElementById("list").addEventListener("click", onClick);

// Même fonction pour pouvoir retirer le listener plus tard
document.getElementById("list").removeEventListener("click", onClick);

// Options : une seule fois | passive (scroll/touch, pas de preventDefault)
link.addEventListener("click", handler, { once: true, passive: true });

// Capture : parent reçoit l’événement avant l’enfant
el.addEventListener("click", handler, true);
  • Ordre : capture (racine → feuille) puis bubbling (feuille → racine).

requestAnimationFrame vs timers 

// Une frame par rafraîchissement écran (~60 Hz) — idéal pour animer le DOM
let id;
function frame() {
  // mettre à jour styles / positions
  id = requestAnimationFrame(frame);
}

Quand quoi ? requestAnimationFrame pour tout ce qui suit le rendu (scroll parallax, canvas, lecture de getBoundingClientRect après layout). setTimeout / setInterval pour la logique métier (polling, délais), pas pour chaque frame d’animation.
